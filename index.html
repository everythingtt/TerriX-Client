How to Use TerriX Executor v3.0

1 First Launch

After installing Tampermonkey and the TerriX Executor script, open the TerriX Client and join a game. Once the game loads, you will see a green TERRIX v3.0 bar at the top of the screen. Click it (or press F2) to open the Executor panel.

If you see "Game Hook Not Detected", you are not using the TerriX Client. Open the TerriX Client URL instead.

2 The Interface

The Executor panel has 5 tabs on the left sidebar:

EDITOR Write and execute JavaScript code with full access to the game's internals via the GameInterface (alias GI), window.G, and TERRIX objects.
LEADERBOARD Live ranking of all players by score (territory × 10 + troops ÷ 50). Your row is highlighted in gold. Updates every 800ms.
SCRIPTS Pre-built scripts: GodBot v3.0, Threat Radar, Auto-Expand, Rush Mode, Team Assist, and Debug Dumper. Click LOAD to insert into the editor.
CONFIG Toggles and numeric inputs for GodBot ratios, ESP overlay, minimap position, and more. All settings persist across sessions.
ESP VIEW Canvas-based extra-sensory perception overlay showing full game state. Enable in CONFIG to activate.

3 Running Scripts

  1. Go to the SCRIPTS tab.
  2. Click LOAD on any script — it gets inserted into the Editor and switches to the EDITOR tab.
  3. Press the green EXECUTE button at the bottom of the panel.
  4. The output area below the editor shows ✓ Executed successfully or the error message.
  5. To stop a looping script (GodBot, Radar, etc.), execute it again — it toggles on/off. Or press STOP ALL.
Most scripts are toggle-based: running them once starts the loop, running them again stops it.

4 GodBot v3.0 — The AI Bot

GodBot is the flagship feature. It automates expansion, attacking, and retreating based on configurable troop-to-territory ratios.

4 Strategies Balanced — expand + attack neighbors. Aggressive — prioritize attacks. Defensive — hold territory. Rush — all-in on weakest player.
Smart Retreat Automatically retreats when troops fall below the retreat ratio × territory. Re-engages when strong enough.
Attack Queuing Sends attacks with a configurable delay between them to avoid the game's rate limiter.
Neighbor Targeting Only attacks enemies that border your territory (toggleable). Sorts targets by territory size — weakest first.

Open the CONFIG tab to customize all GodBot parameters. Key settings:

Expand RatioTroops must exceed territory × this value to expand into neutral land (default: 2.0)
Attack RatioTroops must exceed territory × this value to attack enemies (default: 4.0)
Retreat RatioRetreat when troops fall below territory × this value (default: 0.3)
Tick RateHow often the bot makes a decision, in milliseconds (default: 800ms)
Queue DelayDelay between queued attacks in milliseconds (default: 150ms)

5 Threat Radar

Load and execute the Threat Radar script. It monitors all incoming ships every 300ms. When a ship targets your territory, the panel header flashes red and a warning is logged to the console. Useful as an early-warning system while multitasking.

6 Live Minimap

Enable the minimap in the CONFIG tab. A canvas overlay appears showing:

  • Green — Your territory
  • Red — Enemy territory
  • Blue — Allied/teammate territory

Adjustable: size (px), opacity, position (4 corners), and refresh rate (200ms).

7 Code Executor & GameInterface API

The Editor gives you a JavaScript REPL with these globals:

GThe raw game hook — direct access to all game objects (G.ag, G.aD, G.ac, etc.)
GIGameInterface — safe wrapper with defensive property access and fallbacks
TERRIXThe Terrix namespace — config, loops registry, version
LoggerPrefix-tagged console logger ([TerriX])

Common GI methods:

GI.myIdYour player ID
GI.getMyTroops()Your current troop count
GI.getMyTerritory()Your territory tile count
GI.getPlayerTerritory(id)Territory count for any player
GI.isPlayerAlive(id)Whether a player is alive
GI.getPlayerName(id)Player name string
GI.areAllies(p1, p2)Alliance check
GI.sendAttack(intensity, targetId)Attack a player
GI.sendAttackTile(intensity, tile, targetId)Attack a specific tile
GI.retreat()Retreat all troops
GI.getBorderTiles(id)Array of border tile encodings for a player
GI.getNeighbors(tile)Array of 4 adjacent tile encodings
GI.getTileOwner(encoded)Owner player ID of a tile
GI.isNeutral(encoded)Whether a tile is unowned
GI.tileToXY(encoded)Convert tile encoding to {x, y} coordinates
Use the Debug Dumper script to inspect all game object properties in the console. Essential for writing custom scripts.

8 Keyboard Shortcuts

F2Toggle the Executor panel open/closed
Ctrl + Shift + XToggle the Executor panel open/closed

9 Multi-Tab Sync

Open multiple TerriX Client tabs simultaneously. They automatically discover each other via BroadcastChannel and sync attack/retreat/peace actions. Enable in CONFIG → Multi-Tab Settings. Each tab shows its peer count in the console on join.

10 Troubleshooting

"Hook detection failed after 20s"You are not on the TerriX Client page. Open the correct URL.
"Cannot read properties of undefined"The game updated and property names changed. Run the Debug Dumper script to check current names.
Script does nothingCheck the output area below the editor for errors. Make sure you are in a game (not the menu).
Panel position lostDrag the header to reposition. Position is auto-saved every 5 seconds.
Config not savingTampermonkey requires GM_setValue grant. Reinstall the script if needed.
